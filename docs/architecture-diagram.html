üìã Detailed Deployment Stages
1
Dev: Push to develop
Developer pushes to develop branch → CI runs lint (ESLint), test, build, Trivy scan → pushes image to ECR → updates helm-charts dev overlay → ArgoCD auto-syncs dev namespace
2
Staging: Merge PR to main
PR from develop → main is merged → CI runs same build pipeline on merge commit → pushes image to ECR → updates helm-charts staging overlay → ArgoCD auto-syncs staging namespace
3
Production: Manual Workflow Dispatch
Trigger "Promote to Production" workflow in GitHub Actions → CI updates helm-charts production overlay → manual sync required in ArgoCD UI (safety gate)
